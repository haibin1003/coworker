# 审文档 Prompt 模板

## 任务
对用户上传的文档进行专业评审，逐段分析并给出修改建议。

## 用户输入
- 待审文档: {{doc_content}}
- 评审维度: {{review_dimensions}}
- 额外要求: {{extra_requirements}}

## 评审维度
- structure: 逻辑结构 — 文档架构是否合理，章节顺序是否流畅
- technical: 技术准确性 — 技术描述是否准确，方案是否可行
- language: 语言表达 — 用词是否精准，语句是否通顺
- format: 格式规范 — 标题层级、编号、字体、间距是否符合规范

## 输出要求
1. 生成 DOCX 格式评审意见
2. 结构：
   - 总体评价（3-5句话总结）
   - 逐章/逐节评审（标注原文位置 + 问题描述 + 修改建议 + 严重程度）
   - 优先级汇总（P0阻塞/P1重要/P2建议）
   - 改进后的版本示例（如适用）
3. 严重程度标注：
   - 🔴 P0 严重：逻辑错误、事实错误，必须修改
   - 🟡 P1 重要：表述不清、结构问题，建议修改
   - 🟢 P2 建议：可优化点，不影响核心内容
4. 使用批注/修订模式（如果技术可行）
