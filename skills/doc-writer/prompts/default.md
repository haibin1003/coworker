# 写文档 Prompt 模板

## 任务
根据用户提供的主题和大纲，生成一份结构化的专业文档。

## 用户输入
- 文档标题: {{title}}
- 文档大纲: {{outline}}
- 文档风格: {{style}} （formal=正式技术文档, concise=简明方案, plain=通俗易懂）
- 参考文件: {{ref_files}}
- 设计系统: {{design_system}}

## 设计系统规范
如用户选择了设计系统，你必须严格遵循下方 DESIGN.md 中的：
- 色彩方案（精确 Hex 值用于标题、正文、强调色）
- 字体组合（标题用 Serif/Sans，正文用什么字体和字重）
- 间距与布局（段间距、页边距、标题层级样式）
- 整体视觉氛围（专业/温暖/科技/极简）

设计系统内容如下：
```
{{design_system_content}}
```

## 输出要求
1. 生成 DOCX 格式文档
2. 根据大纲自动展开每章节内容，每章 200-500 字
3. 风格指南：
   - formal: 使用规范的技术术语，段落结构清晰，适当使用编号和列表
   - concise: 开门见山，每段3-4句，多用要点列表
   - plain: 避免术语，用日常语言解释，适合非专业人士阅读
4. 如果选择了设计系统，文档的标题颜色、字体、间距应遵循其规范
5. 包含：封面（标题+日期）、目录、正文、附录（如有）
6. 文件命名：{{title}}.docx
