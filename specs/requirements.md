# Co-Worker 需求规范

> 基于与用户(孙工)的多轮讨论整理。最后更新: 2026-05-15

---

## 1. 项目愿景

做一个普通人也能用的 AI 办公工具。用户不需要知道 AI 是什么，点按钮就能完成写周报、做PPT、写专利等办公任务。

## 2. 用户故事

### 普通用户
- **US-01** 作为办公室白领，我想写一份周报，不需要学习 prompt，填几个关键信息就能生成
- **US-02** 作为产品经理，我想把产品方案快速做成 PPT，拖入文档就能自动排版
- **US-03** 作为专利工程师，我想从想法/文档/代码出发，快速生成专利交底书
- **US-04** 作为行政人员，我想整理散乱的文件，AI 自动分类归档
- **US-05** 作为普通员工，我想写一封正式邮件，只需提供收件人和要点

### 技术用户
- **US-06** 作为开发者，我想切换到高级模式，自由使用 Claude Code 的所有能力
- **US-07** 作为架构师，我想让 AI 帮我做需求分析、方案设计和评审

## 3. 功能需求

### FR-01：任务面板（OfficePanel）
- 展示技能卡片网格，按分类分组
- 支持搜索技能
- 最近使用记录
- 点击卡片进入技能向导

### FR-02：技能向导（TaskWizard）
- 根据技能定义动态渲染表单
- 支持文件拖拽上传
- 支持输出格式选择
- "一键生成"按钮 → 组装 prompt → 调用 agent

### FR-03：流式进度（ProgressView）
- 非终端日志，是分步骤的进度展示
- "正在分析..."→"正在生成..."→"完成！"
- 每个步骤有状态（等待/进行中/完成/失败）

### FR-04：结果展示（ResultView）
- 展示生成的文件（可打开/预览）
- 👍/👎 评分
- 可选文字反馈
- 反馈写入技能反馈系统

### FR-05：专利技能 ★核心
- 三种输入模式：天马行空/文档/代码
- 专利网初筛（快速查新+方向评估）
- 标准 8 步流程（复用 patent-disclosure skill）
- 三文档输出（检索报告+交底书+调研分析）

### FR-06：高级模式
- 保留 Open Cowork ChatView 完整功能
- 模式切换不丢失状态
- 高级模式可手动调用普通模式的技能

### FR-07：首次启动引导
- 配 API Key（默认推荐 DeepSeek）
- 选工作区
- 快速体验（一个示例任务）

### FR-08：技能自进化
- 累计 10 条反馈触发优化
- AI 分析反馈 → 生成新版本 → A/B 测试
- 新版更好？升级 : 回退

## 4. 非功能需求

- **NFR-01** Windows .exe 一键安装
- **NFR-02** 默认模型 DeepSeek（成本低、中文好）
- **NFR-03** MIT 开源协议
- **NFR-04** 普通用户 30 秒内完成第一个任务
- **NFR-05** GUI 响应流畅，不卡顿
- **NFR-06** 基于 Open Cowork Fork，只改 Renderer 层

## 5. MVP 范围

6 个技能：写周报、写文档、做PPT、写专利、审文档、写邮件
