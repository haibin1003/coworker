# Co-Worker 产品设计文档 v1.0

> **Co-Worker**：协同办公桌面应用。把 Claude Code 包装成普通人点按钮就能用的办公工具。
>
> 状态：待评审 | 日期：2026-05-15

---

## 1. 产品定位

### 1.1 一句话

**Co-Worker = 可视化的 Claude Code，把办公场景变成一键可用的技能按钮。**

### 1.2 不是 / 是

不是 AI 聊天工具（ChatGPT）、不是 AI 编程助手（Cursor）、不是通用 Agent 平台（Open Cowork）。
是一个面对**真实办公场景**的桌面应用。每个场景是一个打磨过的技能，有专属的引导表单。

### 1.3 核心差异化

| 维度 | ChatGPT | Open Cowork | Co-Worker |
|------|---------|-------------|-----------|
| 入口 | 空白输入框 | 空白输入框 | **场景按钮 + 引导表单** |
| 用户心智 | "我能问什么？" | "我能让它做什么？" | **"我想做这件事"** |
| 输出质量 | 依赖 prompt 质量 | 依赖 prompt 质量 | **技能模板保证下限** |
| 技能进化 | 无 | 无 | **用户反馈驱动的自进化** |

## 2. 用户模型

**双层用户**：
- **普通用户（80%）**：办公室白领、产品经理、专利工程师。不需要知道 AI 是什么。
- **技术用户（20%）**：开发者、架构师。可切换到高级模式（ChatView）。

**设计原则**：
1. 零学习成本：普通用户 30 秒内完成第一个任务
2. 渐进式暴露：高级功能放在高级模式
3. 结果导向：看到的是"生成的文件"，不是"AI 的思考过程"

## 4. 专利技能深度设计 ★核心

### 4.1 三种输入模式

**模式一：💡 天马行空（创意输入）**
用户写一段话描述想法 → AI 头脑风暴 → 列出 3-5 个候选专利方向 → 专利网初筛（快速查新）→ 用户选择方向 → 进入标准 8 步流程

**模式二：📁 给文档**
上传文档（pdf/docx/md/txt/pptx）→ AI 提取技术要点 → 列出候选专利点 → 用户确认 → 进入标准 8 步流程

**模式三：💻 给代码**
选择项目目录 → AI 扫描代码结构 → 识别核心算法/创新架构 → 列出候选专利点（附代码引用）→ 用户选择 → 进入标准 8 步流程

### 4.2 标准 8 步流程

```
Step 1: 边界确认
Step 2: 项目扫描 / 技术分析
Step 3: 专利点挖掘
Step 4: 融合为方法+系统专利
Step 5: 国知局查新（三层降级：cnipa → delegate子代理 → 训练知识）
Step 6: 摘要预览 → 用户确认
Step 7: 生成全文交底书（中国移动模板，九章）
Step 8: 自检（GitHub暴露/URL脱敏/代号泛化/权利要求上位化）
```

**输出三份文档**：检索报告 + 技术交底书 + 调研分析（.md + .docx 双格式）

### 4.3 与现有 patent-disclosure Skill 的集成

Co-Worker 的专利技能是对现有 `patent-disclosure` skill（v1.9.0）的 GUI 产品化封装。复用其 8 步流程、查新能力、模板生成，新增天马行空输入模式和专利网初筛。

## 6. 技术架构


### 6.4 最大化复用 Open Cowork 原生能力

Co-Worker 不是替代 Open Cowork，而是在其之上叠加办公场景层。所有原生能力全部透传到技能中：

| 原生能力 | 透传方式 | 用户看到的效果 |
|---------|---------|-------------|
| **Skills 系统** | 我们的 24 个技能与自带 pptx/docx/xlsx/pdf 混排 | 用户分不出哪个是原生哪个是新增 |
| **MCP 连接器** | TaskWizard 加"联网搜索/Notion同步"开关 | 写文档时自动附带网络资料 |
| **Sandbox 沙箱** | 所有文件操作自动走 WSL2/Lima | 用户零感知，安全兜底 |
| **记忆系统** | 每个技能读写用户偏好到 memory | 二次使用自动适配用户风格 |
| **定时任务** | OfficePanel 技能卡片加"定时"按钮 | "每周五自动生成周报草稿" |
| **远程控制** | 技能注册到 feishu/slack channel | 手机发"写周报"，桌面端执行 |
| **GUI 自动化** | 专利/PPT 技能调用 screen capture | 生成后自动打开 Office 调格式 |
| **工作区管理** | 每个技能 session 绑定工作区 | 文件拖入即用，自动隔离 |

#### 透传架构

```
OfficePanel (技能卡片)
    │ 用户点击技能
    ▼
TaskWizard (引导表单)
    ├── 普通字段：文本/选型/文件
    ├── MCP开关：☑ 联网搜索  ☑ Notion同步     ← 透传 MCP
    ├── 定时设置：☑ 设为定时任务 [每周五 17:00] ← 透传定时任务
    └── 记忆选项：☑ 记住我的偏好               ← 透传记忆系统
    │
    ▼
Skill Bridge
    ├── 组装 prompt（含用户输入 + 记忆注入 + MCP 上下文）
    ├── 通过 IPC → agent-runner（不改）
    │     ├── Sandbox 自动隔离文件操作
    │     ├── Skills 引擎执行工具调用
    │     ├── MCP 工具按开关启用
    │     └── GUI 自动化按需调用
    │
    ▼
ResultView
    └── 评分 → 写入 feedbackStore → 触发进化
```

#### 定时任务集成

技能卡片上的"定时"按钮，利用 Open Cowork 的 ScheduledTaskManager：

```typescript
// 用户点"定时" → 弹出 Cron 设置
// → 创建 ScheduledTask，action = "execute_skill"
// → 到时间自动调 agent-runner 执行技能
```

#### 远程控制集成

技能注册到 Open Cowork 的 remote-manager：

```typescript
// 飞书/Slack 收到"写周报"指令
// → message-router 匹配技能名称
// → 自动调 TaskWizard → agent-runner
// → 结果文件回传飞书
```

## 7. 双模交互

- **普通模式（默认）**：任务面板 → 点技能 → 填表单 → 等结果。不需要知道 AI 是什么。
- **高级模式**：完整 ChatView（Open Cowork 原有），文件树、MCP 工具、自由对话。
- **切换**：顶栏按钮，切换时保持状态不丢失。

## 8. 技能自进化系统

```
用户使用 → 评分 👍/👎 → 累计10条反馈 → AI分析
→ 优化prompt/表单 → 生成新版本 → A/B测试
→ 新版更好？升级为主版本 : 回退
```

进化触发条件：累计 10 条反馈 或 连续 3 条负面 或 用户主动触发。
